# Git 常用三连命令详解

这三个命令是 Git 本地工作流的核心，**修改代码后上传到 GitHub 的必经之路**。

---

## 1. `git add .`

- **命令**：`git add .`
- **含义**：将当前目录下的**所有改动**（新增、修改、删除的文件）添加到**暂存区（Staging Area）**。
- **作用**：相当于告诉 Git：“这些文件是我这次想要记录的变化，请准备拍照。”
- **通俗理解**：你选好了要打包的东西，放在打包台上，但还没打包。
- **注意**：那个点 `.` 代表当前目录所有文件。如果只想添加某个文件，可以写 `git add 文件名`。

---

## 2. `git commit -m "修改了 header 样式"`

- **命令**：`git commit -m "提交说明"`
- **含义**：把暂存区里的内容**永久保存**成一个历史版本（快照），并附上备注。
- **作用**：创建一个新的提交记录，`-m` 后面双引号里的文字就是这次改动的描述。
- **理解**：相当于按下快门，拍了一张代码快照，并写上标签“修改了 header 样式”，以后随时可以翻看或回到这个状态。
- **重要**：提交信息一定要写清楚干了什么，方便自己和别人以后查阅。

---

## 3. `git push`

- **命令**：`git push`
- **含义**：把本地仓库的提交记录**上传**到远程仓库（如 GitHub）。
- **作用**：将本地的所有新提交同步到云端，让其他人看到你的改动，也起到备份作用。
- **理解**：把刚才拍好并贴好标签的快照，发送到 GitHub 的仓库里。
- **前提**：之前已经用 `git remote add cs-map-creater 仓库地址` 绑定过远程仓库，并且首次 push 时用了 `git push -u cs-map-creater main`，后续才能直接用 `git push`。
