# .gitignore 新手完全指南：让 Git 别管不该管的文件

> **一句话理解**：`.gitignore` 是一个"请忽略这些文件"的清单。你告诉 Git：
> "这些文件/文件夹不要跟踪、不要提交、不要上传到 GitHub。"
> **注意事项**：本教程中的touch等命令如果不可用，可以搜索教程安装相关依赖，可以上网查找一份.gitignore的文档模板，或使用AI生成（告诉AI你的使用场景）。

---

## 为什么要学 .gitignore？

先看一个反面案例：

你写了一个 Node.js 项目，执行 `git add .` 的时候，不小心把 `node_modules/`（几万个依赖文件）也加进去了。后果是：

- 仓库体积暴涨几百 MB
- 提交速度变慢
- GitHub 上一片混乱
- 队友拉代码时也得下载这些垃圾文件

**`.gitignore` 就是来解决这个问题的**——提前告诉 Git："这些文件你假装没看见。"

---

## 哪些文件应该被忽略？

在写规则之前，先搞清楚**什么该忽略**：

| 类型 | 举例 | 为什么要忽略 |
|------|------|-------------|
| 依赖包目录 | `node_modules/`、`vendor/`、`venv/` | 别人可以自己装，不需要你传上去 |
| 编译产物 | `dist/`、`build/`、`*.class`、`*.o` | 这些是从源码生成的，有源码就能重新生成 |
| 系统自动生成的文件 | `.DS_Store`（Mac）、`Thumbs.db`（Windows） | 和项目无关，每台电脑不一样 |
| 私密信息 | `.env`、`config.json`（含密码） | **绝对不能上传**，泄露就出大事 |
| 编辑器/IDE 配置 | `.idea/`、`.vscode/`、`*.swp` | 你队友可能用不同的编辑器 |
| 日志和缓存 | `*.log`、`cache/`、`__pycache__/` | 运行时产生的临时文件 |

## 怎么创建 .gitignore？

### 方法一：命令行创建（推荐）

```bash
# 在项目根目录下，创建一个叫 .gitignore 的文件
touch .gitignore
```

> 这个文件名**前面有个点**，在 Mac/Linux 上是隐藏文件。
> 文件名必须**精确**叫 `.gitignore`，少一个点、多一个字母都不行。

### 方法二：用编辑器创建

在 VS Code 里，右键项目根目录 → 新建文件 → 输入 `.gitignore`。

### 创建完之后

```bash
# 把 .gitignore 本身也提交到仓库里，这样队友也能共享这个规则
git add .gitignore
git commit -m "添加 .gitignore 文件"
```

> **注意**：`.gitignore` 本身**不应该被忽略**，它要提交到仓库里让所有人都用同一套规则。

## .gitignore 语法规则（核心重点）

> `.gitignore` 里面每一行写一条规则，Git 会根据这些规则决定忽略哪些文件。

---

### 规则 1：忽略某个具体文件

```
文件名
```

**示例：**

```
config.json
```

**效果**：项目根目录下的 `config.json` 被忽略。

---

### 规则 2：忽略某个文件夹

```
文件夹名/
```

**注意**：文件夹名后面要加 `/`，表示"这是一个文件夹"。

**示例：**

```
node_modules/
```

**效果**：项目里所有的 `node_modules/` 文件夹都会被忽略。

---

### 规则 3：用 `*` 通配符匹配一类文件

`*` 可以代表任意字符（类似"不管叫什么名字都行"）。

**示例：**

```
*.log
```

**效果**：忽略所有以 `.log` 结尾的文件，比如 `error.log`、`server.log`、`debug.log`。

**更多例子：**

| 规则 | 含义 |
|------|------|
| `*.pyc` | 忽略所有 `.pyc` 文件（Python 编译缓存） |
| `*.class` | 忽略所有 `.class` 文件（Java 编译产物） |
| `*.png` | 忽略所有 `.png` 图片 |
| `temp*` | 忽略所有名字以 `temp` 开头的文件 |

---

### 规则 4：用 `**` 匹配任意层级的目录

`**` 表示"不管嵌套了多少层"。

**示例：**

```
**/logs
```

**效果**：忽略**任何位置**叫 `logs` 的文件或文件夹。不管是根目录的 `logs/`，还是 `src/logs/`，还是 `a/b/c/logs/`，全都被忽略。

**另一个常见用法：**

```
**/node_modules/
```

**效果**：忽略项目中所有层级的 `node_modules/` 目录。

---

### 规则 5：用 `!` 取反（不忽略某个文件）

有时候你想忽略一整类文件，但其中某一个你又想保留，就可以用 `!`。

**示例：**

```
*.md
!README.md
```

**效果**：忽略所有 `.md` 文件，**但 `README.md` 除外**（不忽略它）。

> **注意**：`!` 取反只能"重新纳入"已经被忽略的文件夹里的文件。
> 比如你先忽略了 `build/` 整个文件夹，想用 `!build/important.txt` 把里面某个文件捞回来是**不行的**。
> 需要改成：
> ```
> build/*
> !build/important.txt
> ```
> 用 `build/*` 而不是 `build/`，这样 Git 仍然会进入这个文件夹去检查，才能被 `!` 取反。

---

### 规则 6：用 `#` 写注释

以 `#` 开头的行是注释，Git 会忽略它。用来给规则分组、加说明。

**示例：**

```
# 这是注释，Git 不会读这一行
node_modules/
```

---

### 规则 7：用 `/` 指定根目录

不加 `/` 的规则可能匹配任意层级。加上前缀 `/` 就只匹配**项目根目录**下的文件。

**示例：**

```
/test.log
```

**效果**：只忽略根目录下的 `test.log`，不会忽略 `src/test.log`。

---

### 语法速查表

| 写法 | 含义 | 示例 |
|------|------|------|
| `文件名` | 忽略某个具体文件 | `config.json` |
| `文件夹名/` | 忽略某个文件夹 | `node_modules/` |
| `*.后缀` | 忽略某类文件 | `*.log` |
| `前缀*` | 忽略以某字符开头的文件 | `temp*` |
| `**/名字` | 忽略任意层级下的目标 | `**/logs` |
| `/文件名` | 只忽略根目录下的文件 | `/test.log` |
| `!名字` | 不忽略（取反） | `!README.md` |
| `#` | 注释 | `# 依赖包` |

## 重要提醒：.gitignore 只对"还没被跟踪的文件"生效

这是一个**新手最容易踩的坑**。

### 什么意思？

如果你的文件**已经被 Git 跟踪过了**（之前已经 `git add` + `git commit` 过），那么后来再写进 `.gitignore` 是**没用的**，Git 还是会继续追踪它。

### 举个例子

```
步骤 1：你提交了 .env 文件（提交了 = Git 已经在跟踪它了）
步骤 2：后来你把 .env 写进了 .gitignore
步骤 3：你改了 .env 的内容
步骤 4：git status → .env 还是会出现在修改列表里！！！
```

### 怎么解决？

需要**先让 Git 停止跟踪这个文件**，再用 `.gitignore` 忽略它：

```bash
# 从 Git 的跟踪列表中移除这个文件（但本地文件不会被删除）
git rm --cached 文件名

# 如果是文件夹，加上 -r
git rm --cached -r 文件夹名/

# 然后提交这个"移除"操作
git add .
git commit -m "从跟踪中移除 xxx 文件"
```

| 参数 | 含义 |
|------|------|
| `git rm` | 从 Git 中移除文件 |
| `--cached` | **只从 Git 跟踪列表中移除**，你电脑上的文件不会被删除（不加这个参数会把文件也删掉！） |
| `-r` | 递归，用于文件夹 |

> **以后再往 `.gitignore` 里加规则就生效了。**

## 完整操作示例：从零开始设置 .gitignore

假设你有一个前端项目，想忽略 `node_modules/`、`.env`、`.DS_Store`：

```bash
# 1. 进入项目目录
cd my-project

# 2. 创建 .gitignore 文件
touch .gitignore

# 3. 用编辑器打开，写入以下内容（也可以用命令行写入）
```

**.gitignore 内容：**

```gitignore
node_modules/
.env
.DS_Store
dist/
```

```bash
# 4. 把 .gitignore 本身提交到仓库
git add .gitignore
git commit -m "添加 .gitignore 规则"

# 5. 以后正常写代码、提交，Git 会自动忽略上面列出的文件
git add .
git commit -m "写完了登录页面"
git push
```

> **注意**：`node_modules/` 之类的文件夹永远不会出现在你的提交里了。

## 常见问题 FAQ

### Q1：`.gitignore` 里能不能写中文注释？

可以，但建议文件用 **UTF-8 编码**保存，否则可能乱码。

### Q2：一个项目可以有多个 `.gitignore` 吗？

可以。**每个子文件夹里都可以放一个 `.gitignore`**，里面的规则只对该文件夹及其子目录生效。但一般项目只需要在根目录放一个就够了。

### Q3：`.gitignore` 和 `.git/info/exclude` 有什么区别？

两者功能一样，但 `.gitignore` 会被提交到仓库里共享给所有人，而 `.git/info/exclude` 是你本地的私有规则，不会被提交。**一般用 `.gitignore` 就够了。**

### Q4：GitHub 上创建仓库时勾选了 "Add a README"，会自动生成 `.gitignore` 吗？

勾选 "Add a README" 不会。但创建仓库时有一个下拉菜单 **"Add .gitignore"**，可以选择语言模板，GitHub 会自动帮你生成一份。

### Q5：我想忽略所有文件，只跟踪某几个文件怎么写？

```gitignore
# 忽略所有文件
*

# 但不忽略这些（白名单）
!.gitignore
!README.md
!src/
!src/**
```
