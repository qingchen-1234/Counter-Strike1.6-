# CSMapCollab (CS 1.6 Web 地图编辑器) 后续开发规划路线图

基于 Vue 3 + Three.js + Socket.io 的 Stateless 实时协同架构已初步完成。核心三视图漫游、ConvexGeometry 不规则凸多边形支持、实时协同数据流已打通。

本路线图（Roadmap）确立了项目接下来的核心开发阶段，确保项目从“可用”向“好用”，最终达到“生产力工具”级别的蜕变。

---

## 阶段一：完美的 .map 生态兼容与无损 I/O (最高优先级)
**目标**：打破生态壁垒，允许直接读取并修改现有 CS 1.6 社区的成熟作品，并保证数据的无损流转。

- [ ] **全量读取第三方 `.map` 文件**
  - 增强 `.map` 解析器，除了识别 `worldspawn` 和基础 Brush（画刷），还要能够存储现有的纹理属性、UV 数据和未知的实体数据。
- [ ] **无损原样导出机制 (Lossless Export)**
  - 核心原则：**不认识的数据绝不破坏**。
  - 在导入时，将未修改的几何体原始字符串、属性、实体完整缓存下来。
  - 在导出时，对于用户在编辑器中新建/修改的部分按照我们标准的算法导出；对于未触碰的原有数据，根据导入时的格式“原封不动”地写回文件。

---

## 阶段二：高级网格编辑工具 (Advanced Brush Tools)
**目标**：赋予玩家真正意义上的地图塑形能力，而不仅限于摆放基础几何体。

- [ ] **网格吸附开关 (Grid Snapping Toggle)**
  - 增加网格吸附功能及 UI 上的全局开关控制。
  - 确保平移、缩放和绘制时，顶点严格对齐网格，避免 BSP 编译时产生 Leak（泄露）。
- [ ] **顶点编辑工具 (Vertex Tool)**
  - 允许在 3D/2D 视图中单独或框选多个顶点进行拖拽。
  - 动态重构 `ConvexGeometry` 形状，支持复杂不规则地形的制作。
- [ ] **平面切割工具 (Clipping Tool)**
  - 实现经典的两点/三点平面切割逻辑。
  - 通过切割面计算将一个 Brush 一分为二，并保留各自的凸多边形特性。

---

## 阶段三：编辑器交互与 UX 升级 (Editor UX/UI)
**目标**：提供符合直觉且高效的现代编辑器操作体验。

- [ ] **快捷键与操作分离优化**
  - **WASD**：严格绑定为视图基础漫游按键（3D 飞行、2D 视野平移）。
  - **方向键 (Up/Down/Left/Right)**：绑定为对当前**已选中方块**在 2D 视图下的精准平移（Nudge）。
- [ ] **现代 3D 变换控件 (TransformControls)**
  - 引入并适配 Three.js `TransformControls`。
  - 为已选对象提供直观的移动、旋转、缩放操作轴 (Gizmo)。
- [ ] **批量选择与操作**
  - 实现鼠标拖拽框选功能。
  - 支持多选合并操作，以及方块的复制 (Copy) / 粘贴 (Paste)，并确保这些操作与 Socket.io 协同广播完美结合。

---

## 阶段四：实体系统支持 (Entity System)
**目标**：让静态的世界“活”起来，实现重生点、灯光与交互机关。

- [ ] **数据结构与 UI 面板区分**
  - 在 UI 侧边栏新增“实体 (Entities)”面板。
  - **Point Entities (点实体)**：如 `info_player_start`, `light` 等，在视图中使用 Sprite 或简易 3D 模型占位符表示。
  - **Brush Entities (笔刷实体)**：如 `func_wall`, `trigger_multiple` 等，支持依附并包裹在现有的几何方块上。
- [ ] **实体数据导出**
  - 在 `.map` 导出算法中，将实体数据正确地封装进 `{ }` 语法块中，并写入完整的 KV（Key-Value）属性。

---

## 阶段五：多人协同进阶 (Advanced Collaboration)
**目标**：在多人同时编辑同一张地图时，保证安全、不冲突且支持后悔药。

- [ ] **元素锁定机制 (Locking Mechanism)**
  - 当玩家 A 选中或正在编辑某个方块/实体时，通过 Socket.io 广播该元素的锁状态。
  - 玩家 B 在 UI 上会看到该元素被高亮或锁定提示，且无法对其进行编辑，避免数据覆盖冲突。
- [ ] **协同环境撤销/重做栈 (Undo / Redo)**
  - 实现状态的本地化记录与服务端同步。
  - 支持对意外操作的安全撤回。

---

## 待定/后续规划 (Backlog)
- **纹理与 UV 映射 (Texture & UV Mapping)**：当前以白模/纯色方块为主，后续将视需求引入 `.wad` 解析器、纹理赋予面板及 UV 参数（偏移/缩放/旋转）的计算与编辑支持。
