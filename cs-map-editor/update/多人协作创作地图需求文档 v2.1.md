## 1. 架构总览

### 1.1 技术栈

| 层 | 技术 | 说明 |
|---|------|------|
| 前端 3D | Three.js | 方块渲染、场景交互 |
| 前端 UI | Vue 3 (Composition API) + Vite | 响应式布局 |
| 后端 | Node.js + Express | REST API + WebSocket |
| 通信 | Socket.io | 房间管理、实时同步 |

### 1.2 数据流

```
用户操作 → Vue组件事件 → App.vue (调度中心)
                            ├── BlockManager (数据CRUD)
                            ├── SceneManager (3D渲染)
                            ├── SocketClient (网络发送)
                            └── MapExporter (导出)

服务器 ← Socket.io事件 → 其他客户端
```

### 1.3 已解决的架构问题

| 问题 | 状态 | 解决方式 |
|------|------|----------|
| ID 生成兼容非 HTTPS | ✅ 已解决 | `crypto.randomUUID` → `Date.now()+Math.random()+performance.now` 三段式降级 |
| TransformControls position 未回写 | ✅ 已解决 | 监听 `objectChange` 事件 → `syncBlockFromMesh()` 实时同步 |
| 服务器仅内存存储 | ⚠️ 待优化 | 短期：内存 + 定期备份 JSON |
| 远程方块可见性 | ✅ 已解决 | `room:state` 时自动遍历渲染 + 新增 `block:refresh` 刷新机制 |
| Socket 连接异步 UI 不同步 | ✅ 已解决 | `onConnected`/`onDisconnected` 回调精确控制 RoomPanel 状态 |

### 2.3 房间事件（v2.1 完整版）

| 事件 | 方向 | 说明 |
|------|------|------|
| `room:create` | 客户端→服务器 | 创建房间（含名称、密码） |
| `room:created` | 服务器→客户端 | 创建成功，返回 roomId |
| `room:list` | 客户端→服务器 | 请求房间列表 |
| `room:list-update` | 服务器→客户端 | 推送房间列表 |
| `room:join` | 客户端→服务器 | 加入房间（含房间号、密码） |
| `room:state` | 服务器→客户端 | 房间完整状态（用户、方块、锁） |
| `room:error` | 服务器→客户端 | 错误消息（房间不存在、密码错误等） |
| `room:leave` | 客户端→服务器 | 离开房间 |
| `room:user-joined` | 服务器→其他客户端 | 有新用户加入 |
| `room:user-left` | 服务器→其他客户端 | 有用户离开 |
| `block:refresh` | 客户端→服务器 | 请求重新发送所有方块 |
| `block:refresh-all` | 服务器→客户端 | 发送所有方块（仅给请求者） |

### 3.2 方块类型（已实现）

| 类型 | shape | Three.js 实现 |
|------|-------|---------------|
| 立方体 | `cube` | `BoxGeometry` |
| 斜坡 | `ramp` | Shape → `ExtrudeGeometry` (45° 斜面) |

| 楔形 | `wedge` | 三角形 Shape → `ExtrudeGeometry` |
| 圆柱 | `cylinder` | `CylinderGeometry` (16 分段) |
| 平面 | `plane` | `BoxGeometry(宽, 2, 长)` 薄板 |

### 3.3 方块操作按钮（v2.1 新增）

属性面板底部操作区域：

| 按钮 | 功能 |
|------|------|
| ↻ X90° / Y90° / Z90° | 绕指定轴旋转 90° |
| ↔ X镜像 / Y镜像 / Z镜像 | 沿指定轴镜像翻转位置 |

### 3.4 方块状态展示

| 状态 | 外观 |
|------|------|
| **正常** | 灰色调 + 半透明 (opacity 0.9) |

## 11. 兼容性与安全

### 11.1 已修复的兼容性问题

| 问题 | 解决方式 |
|------|----------|
| `crypto.randomUUID()` 非 HTTPS 不可用 | 三段式降级方案 |
| Socket WebSocket 被阻断 | `transports: ['websocket', 'polling']` 自动降级 |

- 房间最大用户数：16 人 ✅
- 密码传输（明文，待 SHA-256）⚠️
- 单房间最大方块数：无限制 ⚠️（待加）

## 附录 B：Socket 事件速查表（v2.1 完整版）

```
房间:
  room:create      → 创建房间
  room:created     ← 创建成功 (含 roomId)
  room:list        → 请求列表
  room:list-update ← 列表更新
  room:join        → 加入房间
  room:state       ← 房间完整状态
  room:error       ← 错误消息
  room:leave       → 离开
  room:user-joined ← 新用户
  room:user-left   ← 用户离开

方块:
  block:create     → 创建方块
  block:created    ← 方块已创建
  block:update     → 更新方块
  block:updated    ← 方块已更新
  block:delete     → 删除方块
  block:deleted    ← 方块已删除
  block:refresh    → 请求刷新
  block:refresh-all ← 全部方块

锁:
  lock:acquire     → 获取锁
  lock:acquired    ← 锁已获取
  lock:release     → 释放锁
  lock:released    ← 锁已释放
  lock:denied      ← 锁拒绝
  block:locked     ← 方块被锁定

光标:
  cursor:move      → 光标移动
  cursor:moved     ← 光标已移动
```
