## 1. 架构总览

### 1.1 技术栈

| 层 | 技术 | 说明 |
|---|------|------|
| 前端 3D | Three.js | 方块渲染、场景交互、四视图渲染 |
| 前端 UI | Vue 3 (Composition API) + Vite | 响应式布局、视图切换 |
| 后端 | Node.js + Express | REST API + WebSocket + 文件存储 |
| 通信 | Socket.io | 房间管理、实时同步 |
| 存储 | 服务端文件系统 + 浏览器 Blob | 保存/读取 `.csmap` 文件 |

### 1.2 数据流（v2.2 增强）

```
用户操作 → Vue组件事件 → App.vue (调度中心)
                            ├── BlockManager (数据CRUD)
                            ├── SceneManager (3D渲染 × 4视口)
                            ├── SocketClient (网络发送)
                            ├── MapExporter (.map导出)
                            ├── SaveManager (保存/读取 .csmap)  ← 新增
                            └── ViewportManager (四视图管理)   ← 新增

服务器 ← Socket.io事件 → 其他客户端
服务器 ← 文件系统 (data/*.csmap)
```

### 1.3 已解决的架构问题

| 问题 | 状态 | 解决方式 |
|------|------|----------|
| ID 生成兼容非 HTTPS | ✅ | `crypto.randomUUID` → 三段式降级 |
| TransformControls position 未回写 | ✅ | `objectChange` → `syncBlockFromMesh()` 实时同步 |
| 服务器仅内存存储 | ✅ (v2.2) | 新增文件持久化：`cs-map-editor/data/*.csmap` |
| 远程方块可见性 | ✅ | `room:state` 自动渲染 + `block:refresh` 刷新 |
| Socket 连接异步 UI 不同步 | ✅ | `onConnected`/`onDisconnected` 回调 |
| 局域网/外网访问 | ✅ | `host: true` + `allowedHosts: true` + ngrok 支持 |

## 4. 多视角编辑 🆕

> **优先级**: P1 — 与 .map 导出同级
> **目标**: 提供顶视图/前视图/侧视图 3 个正交投影 + 1 个自由透视视角，共 4 个视口

### 4.1 四视图布局

```
┌──────────────────────────────────────────────┐
│  🧱 CSMapCollab           工具栏              │
├────────┬──────────────────┬──────────────────┤
│ 房间面板│                  │   前视图(正交)    │
│        │  自由视角(透视)   │    XZ平面          │
│        │                  │                  │
│        │                  ├──────────────────┤
│        │                  │   侧视图(正交)    │
│        │                  │    YZ平面          │
│        ├──────────────────┴──────────────────┤
│        │   顶视图(正交)                       │
│        │   XY平面 (从上往下看)                 │
├────────┴─────────────────────────────────────┤
│  状态栏                                       │
└──────────────────────────────────────────────┘
```

### 4.2 各视图行为

| 视图 | 投影方式 | 视角方向 | 可旋转 | 可平移 | 可缩放 |
|------|----------|----------|--------|--------|--------|
| **自由视角** | 透视 (Perspective) | 任意 | ✅ WASD + 右键/中键 | ✅ | ✅ 滚轮 |
| **顶视图** | 正交 (Orthographic) | -Z (从上往下) | ❌ | ✅ 中键拖拽 | ✅ 滚轮 |
| **前视图** | 正交 (Orthographic) | -Y (从前往后) | ❌ | ✅ 中键拖拽 | ✅ 滚轮 |
| **侧视图** | 正交 (Orthographic) | -X (从右往左) | ❌ | ✅ 中键拖拽 | ✅ 滚轮 |

### 4.3 正交视图相机配置

```typescript
// 顶视图 (Top): 相机在 Y+ 上方，看向原点，Z轴向上
const topCamera = new THREE.OrthographicCamera(
  -size, size, size, -size, 0.1, 10000
)
topCamera.position.set(0, 2000, 0)
topCamera.lookAt(0, 0, 0)

// 前视图 (Front): 相机在 Z+ 前方，看向原点
const frontCamera = new THREE.OrthographicCamera(
  -size, size, size, -size, 0.1, 10000
)
frontCamera.position.set(0, 0, 2000)
frontCamera.lookAt(0, 0, 0)

// 侧视图 (Side): 相机在 X+ 右侧，看向原点
const sideCamera = new THREE.OrthographicCamera(
  -size, size, size, -size, 0.1, 10000
)
sideCamera.position.set(2000, 0, 0)
sideCamera.lookAt(0, 0, 0)
```

### 4.4 视口交互细节

| 功能 | 说明 |
|------|------|
| 视口激活 | 鼠标点击某个视口即激活，高亮边框指示当前激活视口 |
| TransformControls | 仅自由视角视口显示 Gizmo（正交视图仅用于观察） |
| 方块选中 | 所有视口均可左键点击选中方块 |
| 视口大小 | 可通过拖拽分界线调整各视口占比 |
| 快捷键切换 | `Tab` 在 4 个视口间循环切换焦点 |
| 双击放大 | 双击某个视口 → 该视口临时全屏，再双击恢复 |

### 4.5 技术实现要点

- 使用 4 个独立的 `THREE.WebGLRenderer` 渲染到不同 `<canvas>`（性能最优）
  - 或使用 1 个渲染器 + `setScissor`/`setViewport`（共享 WebGL 上下文）
- 推荐方案：**1 个渲染器 + scissor/viewport 分块**，减少 GPU 内存开销
- 3 个正交相机共享同一个 `scene`，仅 `camera` 不同
- TransformControls 只在透视相机视口激活
- GridHelper 在所有视口可见

> **优先级**: P1 — 与 .map 导出同级
> **目标**: 用户可保存当前地图为 `.csmap` 文件，随时读取恢复编辑

### 5.1 保存流程

```
点击 [💾 保存] 按钮
    │
    ├─ 首次保存？
    │   ├─ 是 → 弹出命名对话框
    │   │         ┌─────────────────────────┐
    │   │         │  💾 保存地图文件          │
    │   │         │                          │
    │   │         │  文件名: [ my_kz_map ]    │
    │   │         │  格式:    .csmap (JSON)   │
    │   │         │                          │

    │   │         └─────────────────────────┘
    │   │
    │   └─ 否 → 直接覆盖上次文件
```

### 5.3 读取流程

```
点击 [📂 打开] 按钮
    │
    ├─ 在线模式 → 弹出文件选择器
    │              ┌────────────────────────────┐
    │              │  📂 打开地图文件             │
    │              │                             │
    │              │  服务器存档:                 │
    │              │  ├─ #4821_KZ_Map_1.csmap    │
    │              │  ├─ #4821_de_dust.csmap     │
    │              │  └─ ...                     │
    │              │                             │
    │              │  或 [从本地选择文件]          │
    │              │                             │
    │              │  ⚠️ 当前方块将被替换          │
    │              │  [取消]        [打开]        │
    │              └────────────────────────────┘
    │
    └─ 离线模式 → 浏览器 `<input type="file">`
                   选择本地 .csmap 文件
            ↓
         读取 JSON → 解析 → 清空场景 → 逐块渲染
```

### 5.4 读取后的渲染流程

```
1. const data = JSON.parse(fileContent)
2. blockManager.blocks.clear()
3. sceneManager.clearAllBlocks()       // 移除所有 mesh
4. for (const blockData of data.blocks) {
     blockManager.createBlock(blockData)
     sceneManager.renderBlock(blockData)
   }
5. 更新 blockCount 显示
6. 如果在线模式 → 广播全部方块给房间其他人
```

## 13. 兼容性与安全

### 13.1 已修复的兼容性问题

| 问题 | 解决方式 |
|------|----------|
| `crypto.randomUUID()` 非 HTTPS 不可用 | 三段式降级方案 |
| Socket WebSocket 被阻断 | `transports: ['websocket', 'polling']` |
| 非 localhost 环境连接 | `detectServerUrl()` 自动检测 |
| Socket 异步连接 UI 不更新 | `onConnected`/`onDisconnected` 回调 |
| 公司 WiFi AP 隔离 | ngrok 隧道 |
| Vite 拦截 ngrok 域名 | `allowedHosts: true` |

### 13.2 安全性

- 房间最大用户数：16 人 ✅
- 密码传输（待 SHA-256）⚠️
- 单房间最大方块数：10,000 ⚠️（待实现上限）
- `.csmap` 文件大小限制：10 MB ⚠️（待实现）

## 附录 B：Socket 事件速查表（v2.2 完整版）

```
房间:
  room:create      → 创建房间
  room:created     ← 创建成功 (含 roomId)
  room:list        → 请求列表
  room:list-update ← 列表更新
  room:join        → 加入房间
  room:state       ← 房间完整状态
  room:error       ← 错误消息
  room:leave       → 离开
  room:user-joined ← 新用户
  room:user-left   ← 用户离开

方块:
  block:create     → 创建方块
  block:created    ← 方块已创建
  block:update     → 更新方块
  block:updated    ← 方块已更新
  block:delete     → 删除方块
  block:deleted    ← 方块已删除
  block:refresh    → 请求刷新
  block:refresh-all ← 全部方块

锁:
  lock:acquire     → 获取锁
  lock:acquired    ← 锁已获取
  lock:release     → 释放锁
  lock:released    ← 锁已释放
  lock:denied      ← 锁拒绝

光标:
  cursor:move      → 光标移动
  cursor:moved     ← 光标已移动

存储 (v2.2):
  POST /api/save        → 保存 .csmap
  GET  /api/files       → 存档列表
  GET  /api/files/:name → 获取存档
  DELETE /api/files/:name → 删除存档
```
