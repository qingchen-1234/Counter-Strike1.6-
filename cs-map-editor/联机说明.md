# CS 地图编辑器 — 联机说明

> 非本地链接，我的测试是在单电脑，多浏览器进行的，测试 OK。但是我从手机就无法进入页面，如何解决？是否别的 PC 也无法进入我的链接？

## 问题概述

这个问题涉及网络配置，核心是让其他设备（手机/其他 PC）能访问你的开发服务器。以下是分析和解决方案。

---

## 代码层面已做的修复

| 文件 | 修复内容 |
|------|----------|
| `server/index.js` | 显式绑定 `0.0.0.0`，启动时打印所有局域网 IP 地址 |

## 🔧 需要手动操作的步骤

### 1. 检查本地 IP 地址

重启服务器后，终端会打印类似信息：

```
192.168.x.x
```

记下这个 `192.168.x.x` 地址。

### 2. 打开 Windows 防火墙端口（最关键！）

以管理员身份打开 PowerShell，执行：

```powershell
netsh advfirewall firewall add rule name="CSMapCollab 5173" dir=in action=allow protocol=TCP localport=5173
netsh advfirewall firewall add rule name="CSMapCollab 3001" dir=in action=allow protocol=TCP localport=3001
```

### 3. 在其他设备上访问

| 设备 | 操作 |
|------|------|
| 同一 WiFi 的其他电脑 | 浏览器访问 `http://192.168.1.100:5173` |
| 手机（同一 WiFi） | 浏览器访问 `http://192.168.1.100:5173` |
| 不同网络 / 外网 | 见下方 Ngrok 方案 |

> ⚠️ 手机**必须**连接同一个 WiFi 网络，且连接的是家庭/专用网络，不是公共 WiFi。

### 4. 验证方法

在你自己的电脑上也测试非 localhost：

```
http://127.0.0.1:5173      → ✅ 本地
http://192.168.x.x:5173    → ✅ 局域网（自己电脑也要能打开）
```

如果第二个打不开，说明防火墙没生效。

### 5. 🌐 外网访问方案（可选）

如果需要让不在同一 WiFi 的人访问，使用 Ngrok 隧道：

```bash
ngrok http 5173
```

Ngrok 会生成一个 `https://xxxx.ngrok.io` 的公开链接，手机/外网设备都能打开。

## 公司 WiFi 网络隔离问题

> 我使用公司 WiFi，但是手机无法进入，我尝试了 WLAN 的 IP、本地连接的 IP，都无法进入。

### 原因分析

公司 WiFi 最常见的情况是 **AP 隔离**（无线客户端隔离），即同一 WiFi 下的设备之间无法互访——这是企业网络安全策略。这不是代码能解决的问题。

```
你的电脑 (192.168.1.100) ──┬── 公司路由器 ── 互联网 (✅)
                           ┃
手机 (192.168.1.101) ──────┘
                           ← 路由器隔离：端口不通 (❌)
```

### 解决方案

以下提供 **3 种可行方案**：

---

#### 方案一：Ngrok 隧道（推荐，无需改网络）

Ngrok 通过外网服务器穿透，完全绕过公司网络隔离。

**步骤：**

1. 下载 ngrok（无需安装）
   - 浏览器打开 [https://ngrok.com/download](https://ngrok.com/download)
   - 下载 Windows 版，解压到项目目录
2. 启动 ngrok 隧道（指向 Vite 5173 端口）
   ```bash
   ngrok http 5173
   ```
3. Ngrok 会显示一个公网 URL，例如：
   ```
   Forwarding  https://a1b2c3d4.ngrok-free.app → http://localhost:5173
   ```

然后在手机/任何地方访问这个 `https://a1b2c3d4.ngrok-free.app` 即可。

> Socket.io 通过 Vite 代理自动转发，完全兼容。

---

#### 方案二：手机开热点（最简单）

1. 手机开启个人热点
2. 电脑连接手机热点（或通过 USB 共享网络）
3. 电脑上运行 `npm run dev`
4. 手机在同一热点网络下，访问 `http://192.168.x.x:5173`

> 家庭热点没有 AP 隔离，设备之间互通。

---

#### 方案三：Windows 自带移动热点

1. 电脑右下角网络图标 → 右键 → 网络和 Internet 设置
2. 找到 **移动热点** → 开启
3. 手机连接该热点
4. 访问 `http://192.168.137.1:5173`

---

### 验证方法（无论哪种方案）

```bash
http://127.0.0.1:5173      → ✅ 本地
http://192.168.x.x:5173    → ✅ 局域网（自己电脑也要能打开）
```

如果 `192.168.x.x` 自己都打不开，说明防火墙或绑定有问题。执行以下命令重新添加防火墙规则（管理员 PowerShell）：

```powershell
netsh advfirewall firewall add rule name="CSMapCollab 5173" dir=in action=allow protocol=TCP localport=5173
netsh advfirewall firewall add rule name="CSMapCollab 3001" dir=in action=allow protocol=TCP localport=3001
```

---

## 总结

以上三种方案任选一种即可。最推荐 **方案一（Ngrok）**，一次配置长期使用，手机/外网都能访问，且不影响你在公司正常上网。
