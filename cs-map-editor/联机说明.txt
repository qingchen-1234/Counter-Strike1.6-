Analyzed network access issues on company WiFi
公司 WiFi 最常见的情况是 AP 隔离或端口隔离，即同一 WiFi 下的设备之间无法互访（这是企业网络安全策略）。这不是代码能解决的，但有几种替代方案。

Created 1 task
这是公司网络的限制，不是代码问题
公司 WiFi 普遍启用了 AP 隔离（无线客户端隔离），导致：


你的电脑 (192.168.1.100) ──┰── 公司路由器 ── 互联网 (✅)                          ┃手机 (192.168.1.101) ─────┘                          ← 路由器隔离: 端口不通 (❌)
下面给你 3 种可行的解决方案：

方案一：Ngrok 隧道（推荐，无需改网络）
Ngrok 通过外网服务器穿透，完全绕过公司网络隔离。

步骤：


# 1. 下载 ngrok（无需安装）#    浏览器打开 https://ngrok.com/download#    下载 Windows 版，解压到项目目录# 2. 启动 ngrok 隧道（指向 Vite 5173 端口）ngrok http 5173# 3. Ngrok 会显示一个公网 URL，例如：#    Forwarding  https://a1b2c3d4.ngrok-free.app → http://localhost:5173
然后在手机/任何地方访问这个 https://a1b2c3d4.ngrok-free.app 即可。

Socket.io 通过 Vite 代理自动转发，完全兼容。

方案二：手机开热点（最简单）
手机开启个人热点
电脑连接手机热点（或通过 USB 共享网络）
电脑上运行 npm.cmd run dev
手机在同一热点网络下，访问 http://192.168.x.x:5173
家庭热点没有 AP 隔离，设备之间互通。

方案三：Windows 自带移动热点
电脑右下角网络图标 → 右键 → 网络和 Internet 设置
找到移动热点 → 开启
手机连接该热点
访问 http://192.168.137.1:5173
验证方法
无论用哪种方案，先在自己的电脑上确认非 localhost 能访问：


http://127.0.0.1:5173    → ✅ 本地http://192.168.x.x:5173  → ✅ 局域网（自己电脑也要能打开）
如果 192.168.x.x 自己都打不开，说明防火墙或绑定有问题。执行以下命令重新添加防火墙规则（管理员 PowerShell）：


netsh advfirewall firewall add rule name="CSMapCollab 5173" dir=in action=allow protocol=TCP localport=5173netsh advfirewall firewall add rule name="CSMapCollab 3001" dir=in action=allow protocol=TCP localport=3001
建议先试方案二（手机热点），30 秒搞定，不依赖任何外部工具。
